Sistema de antipassback para control de acceso
python flask sqlite

Dentro de un sistema de seguridad, un antipassback es un mecanismo que permite el acceso a una área validando si el usuario tiene permitido o no. Los dispositivos que permiten el acceso, deben estar intercomunicados para poder validar en sus bases de datos el tipo de acceso al usuario, la fecha y hora del último acceso y por último si el usuario está ingresando o saliendo. Es por ello que se dentro del sistema de múltiples dispositivos de acceso, se implementa el patrón de maestro esclavo dentro del sistema, para que sólo uno de ellos posea el control sobre un área, así mismo para realizar la comunicación entre éste como maestro y las demás (esclavas), se implemento un microservicio para realizar esta funcionalidad.

Lectoras biométricas de seguridad
c++ c bash-shell glade sqlite

El cliente requería un sistema de seguridad para puertas que tuviese mútliples tipos de acceso convencional y biométrico: huella,iris, facial,tarjeta,pin.
El dispositivo debía estar vinculado con un sistema que administraba el control de acceso a nivel general : edificios,zonas,grupos de acceso; es por ello que el dispositivo debía contar con algún protocolo de comunicación, control de periféricos externos, acceso a sensores y lectores, además de su propia base de datos.
Dado a lo anterior en el dispositivo se integró un software capaz de controlar periféricos, acceso a sensores, con base de datos ligera para un sistema embebido ,un protocolo de comunicación y una interfaz gráfica.
